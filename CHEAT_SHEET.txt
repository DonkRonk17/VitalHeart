QUICK START
═══════════════════════════════════════════════════════════════════════════════

1. Install dependencies:
   pip install -r requirements.txt

2. Set OLLAMA_KEEP_ALIVE (recommended):
   [System.Environment]::SetEnvironmentVariable("OLLAMA_KEEP_ALIVE", "24h", "User")

3. Run daemon:
   python ollamaguard.py

4. Stop daemon:
   Press Ctrl+C

═══════════════════════════════════════════════════════════════════════════════
CONFIGURATION QUICK EDIT
═══════════════════════════════════════════════════════════════════════════════

Config File: ollamaguard_config.json

Common Changes:

• Change monitored model:
  {"ollama": {"model_name": "your-model"}}

• Adjust check frequency:
  {"ollama": {"check_interval_seconds": 120}}  // Every 2 minutes

• Change inference timeout:
  {"ollama": {"inference_timeout_seconds": 60}}  // More lenient

• Increase restart limits:
  {"restart": {"daily_restart_limit": 20}}

• Disable micro-inference tests (not recommended):
  {"monitoring": {"enable_micro_inference_test": false}}

═══════════════════════════════════════════════════════════════════════════════
HEALTH STATUS MEANINGS
═══════════════════════════════════════════════════════════════════════════════

HEALTHY           ✅  Ollama responsive, model loaded, inference working
MODEL_UNLOADED    ⏳  Model evicted from VRAM (auto-reload triggered)
INFERENCE_FROZEN  ❌  Inference timed out (auto-restart triggered)
SERVER_DOWN       🔴  Cannot connect to Ollama (auto-restart triggered)
GRACE_PERIOD      🔄  Ollama version changed (5-min grace, no interventions)

═══════════════════════════════════════════════════════════════════════════════
RESTART STRATEGY SUMMARY
═══════════════════════════════════════════════════════════════════════════════

Exponential Backoff:
  Attempt 1: 1 second
  Attempt 2: 2 seconds
  Attempt 3: 4 seconds
  Attempt 4+: ESCALATE (no more auto-restarts)

Limits:
  • Max 3 restarts in 5-minute window
  • Max 10 restarts in 24 hours
  • After limits: monitoring-only mode (manual intervention required)

AGENTHEARTBEAT INTEGRATION
═══════════════════════════════════════════════════════════════════════════════

Query Latest Heartbeat:

  from agentheartbeat import AgentHeartbeatMonitor
  monitor = AgentHeartbeatMonitor()
  hb = monitor.get_latest_heartbeat("LUMINA")
  print(hb['status'], hb['metrics']['ollama_version'])

Key Metrics Stored:
  • ollama_version: Ollama version string
  • model_loaded: Boolean (True/False)
  • inference_latency_ms: Inference test duration
  • vram_used_mb: VRAM usage in megabytes
  • restarts_today: Count of restarts in last 24h
  • uptime_hours: Hours since last Ollama restart

═══════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING QUICK FIXES
═══════════════════════════════════════════════════════════════════════════════

Problem: "Connection refused"
Fix: Start Ollama or check if running on different port

Problem: "Model not found"
Fix: ollama pull your-model-name

Problem: "OLLAMA_KEEP_ALIVE not set"
Fix: Set environment variable (see Quick Start #2)

Problem: Daily restart limit exceeded
Fix: Check Ollama logs for crash reasons, consider increasing limit

Problem: Too many false restarts
Fix: Increase inference_timeout_seconds in config

Problem: AgentHeartbeat unavailable
Fix: OllamaGuard will use fallback file (ollamaguard_heartbeat_fallback.jsonl)
